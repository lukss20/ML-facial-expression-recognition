# ML-facial-expression-recognition

## პროექტის აღწერა

ამ პროექტის მიზანია FER2013 (Facial Expression Recognition 2013) მონაცემთა ბაზის გამოყენებით ადამიანის სახის ემოციების კლასიფიკაცია კონვოლუციური ნეირონული ქსელების (CNN) მეშვეობით.

ამოცანა წარმოადგენს მრავალკლასიან კლასიფიკაციას, სადაც მოდელმა 48x48 ზომის შავ-თეთრი გამოსახულებების საფუძველზე უნდა განსაზღვროს ადამიანის ემოციური მდგომარეობა.

პროექტის ფარგლებში გავტესტე რამდენიმე განსხვავებული არქიტექტურისა და ჰიპერპარამეტრების კომბინაციები. ყველა ექსპერიმენტი დავლოგე WandB პლატფორმის გამოყენებით, რისი დახმარებითაც გამიმარტივდა მოდელების სწავლის პროცესის დეტალური ანალიზი და ერთმანეთთან შედარება.

---

# მონაცემთა ბაზა

FER2013 წარმოადგენს ერთ-ერთ ყველაზე პოპულარულ მონაცემთა ბაზას ემოციების ამოცნობის სფეროში.

მონაცემთა ბაზა შეიცავს 48x48 ზომის შავ-თეთრ გამოსახულებებს.

კლასები:

* Angry
* Disgust
* Fear
* Happy
* Sad
* Surprise
* Neutral

# ექსპერიმენტი №1 — TinyCNN (საბაზისო მოდელი)

პირველ ექსპერიმენტში შევქმენი მარტივი CNN არქიტექტურა.

არქიტექტურა:

* Conv(1 → 16)
* MaxPool
* Conv(16 → 32)
* MaxPool
* Fully Connected (128)
* Output Layer (7)

შედეგები:

* Training Accuracy: 65.5%
* Validation Accuracy: 52.63%

ჩემი დასკვნა:

ეს მოდელი გამოვიყენე როგორც საწყისი საბაზისო არქიტექტურა. მიუხედავად იმისა, რომ სასწავლო მონაცემებზე მივიღე შედარებით მაღალი სიზუსტე, ვალიდაციის სიზუსტე ძაან  დაბალი იყო.

ეს მიუთითებს Overfitting-ზე. მოდელმა კარგად დაიმახსოვრა სასწავლო მონაცემები, თუმცა ვერ განაზოგადა ახალი მონაცემები.

---

# ექსპერიმენტი №2 — MediumCNN

TinyCNN-ის გაუმჯობესების მიზნით გავაკეთ უფრო ღრმა არქიტექტურა.

დავამატე:

* მეტი კონვოლუციური ფენა
* Batch Normalization
* Dropout
* მეტი ფილტრი

არქიტექტურა:

* 32 არხი
* 64 არხი
* 128 არხი
* BatchNorm
* Dropout 0.3

შედეგები:

* Training Accuracy: 69.9%
* Validation Accuracy: 60.55%

ჩემი დასკვნა:

Batch Normalization-მა მნიშვნელოვნად დაასტაბილურა, ხოლო Dropout-მა შეამცირა დაზეპირება.

TinyCNN-თან შედარებით იყო დაახლოებით 8%-იანი უკეტესი შედეგი.

---

# ექსპერიმენტი №3 — Underfitting-ის მაგალითი

გამოყენებული იყო:

* Dropout = 0.5
* Weight Decay = 1e-4

შედეგები:

* Training Accuracy: 55.6%
* Validation Accuracy: 54.8%

ჩემი დასკვნა:

ამ შემთხვევაში რეგულარიზაცია აღმოჩნდა ზედმეტად ძლიერი.

მოდელმა ვერ მოახერხა მონაცემებში არსებული მნიშვნელოვანი შაბლონების სწავლა, რის შედეგადაც როგორც თრეინინგ, ასევე ვალიდაციის სიზუსტე დაბალი დარჩა.

ეს წარმოადგენს Underfitting-ის მაგალითს.

---

# ექსპერიმენტი №4 — მონაცემთა აგმენტაცია

გამოვიყენე:

* RandomHorizontalFlip
* RandomRotation

მიზანი იყო მოდელის განზოგადების უნარის გაუმჯობესება.

შედეგები:

* Validation Accuracy ≈ 60%

ჩემი დასკვნა:

აგმენტაციამ შეამცირა Overfitting-ის ეფექტი და გააუმჯობესა მოდელის მდგრადობა სხვადასხვა პოზისა და კუთხის მიმართ.

---

# ექსპერიმენტი №5 — MediumCNN + Scheduler

გამოვიყენე StepLR Scheduler.

პარამეტრები:

* Learning Rate = 0.001
* Step Size = 10
* Gamma = 0.5

შედეგები:

* Validation Accuracy: 60.88%

ჩემი დასკვნა:

Learning Rate Scheduler-მა მოდელს მისცა შესაძლებლობა სწავლის ბოლო ეტაპზე უფრო მცირე ნაბიჯებით განეხორციელებინა ოპტიმიზაცია, რამაც გამოიწვია მცირე, მაგრამ მნიშვნელოვანი გაუმჯობესება.

---

# ექსპერიმენტი №6 — DeepCNN

შემდეგ ეტაპზე შევქმენი უფრო ღრმა არქიტექტურა.

გამოვიყენე:

* 32 → 64 → 128 → 256 არხები
* დამატებითი კონვოლუციური ფენები
* Adaptive Average Pooling

შედეგები:

* Best Validation Accuracy: 65.95%

ჩემი დასკვნა:

მოდელის სიღრმის გაზრდამ მნიშვნელოვნად გააუმჯობესა შედეგები. DeepCNN-მა შეძლო უფრო კომპლექსური მახასიათებლების ამოცნობა სახის გამოსახულებებიდან.

MediumCNN-თან შედარებით მივიღე დაახლოებით 5%-იანი უკეთესი შედეგი მაინც.

---

# ექსპერიმენტი №7 — ResidualCNN

საბოლოოდ ყველაზე კარგი შედეგი მივიღე ResidualCNN არქიტექტურით.

მოდელში გამოვიყენე Residual Block-ები და Skip Connection-ები.

Residual Connection-ის მთავარი უპირატესობა არის ის, რომ ღრმა ქსელებში გრადიენტის გავრცელება ხდება უფრო ეფექტურად.

შედეგები:

* Best Validation Accuracy: 66.34%

ჩემი დასკვნა:

ResidualCNN აღმოჩნდა ყველაზე ძლიერი არქიტექტურა.

თუმცა დაფიქსირდა Overfitting-ის ნიშნებიც:

* Training Accuracy ≈ 88.7%
* Validation Accuracy ≈ 66.3%

სასწავლო და ვალიდაციის სიზუსტეებს შორის დიდი სხვაობა მიუთითებს იმაზე, რომ მოდელმა ნაწილობრივ დაიმახსოვრა სასწავლო მონაცემები.

# დასკვნა

პროექტში გამოვიტენე ოთხი განსხვავებული CNN არქიტექტურისა და რამდენიმე ჰიპერპარამეტრული ტესტირება.

ექსპერიმენტებმა ნათლად აჩვენა როგორც Underfitting-ის, ასევე Overfitting-ის შემთხვევები. დადასტურდა, რომ არქიტექტურის სიღრმის გაზრდა და Residual Connection-ების გამოყენება მნიშვნელოვნად აუმჯობესებს FER2013 მონაცემთა ბაზაზე მიღებულ შედეგებს.

საბოლოოდ საუკეთესო შედეგი მიიღო ResidualCNN მოდელმა, რომელმაც ვალიდაციის მონაცემებზე 66.34%, ხოლო ტესტურ მონაცემებზე 66% სიზუსტე აჩვენა.
